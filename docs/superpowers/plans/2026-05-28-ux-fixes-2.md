## Task 2: Room Code Error Feedback

private buildChipSidebar() {
  this.add.text(10, 90, "CHIPS", { fontSize: "11px", color: C.muted })
  let y = 110
  this.room.state.players.forEach((player, id) => {
    const color = id === this.room.sessionId ? C.chip : C.muted
    const t = this.add.text(10, y, "", { fontSize: "13px", color })
    this.chipSidebarTexts.set(id, t)

private refreshChipSidebar() {
  this.room.state.players.forEach((player, id) => {
    const t = this.chipSidebarTexts.get(id)
    if (!t) return
    const chips = player.chips
    const dots = chips > 0 ? "●".repeat(chips) : "—"
    t.setText(`${player.name}: ${dots} (${chips})`)
  })
}

## Task 6: Fix 2-Player Pawn Direction

**Problem:** `buildPawnDirs` in ChessScene uses `cornerDirs = [-1, -1, 1, 1]` for all player counts. In 2P chess, player index 1 (black, top) gets `-1` (upward) instead of `+1` (downward) — black pawns cannot advance.

**Fix:** Use separate direction arrays for 2P vs 4P.

**Files:**
- Modify: `client/src/scenes/ChessScene.ts`

- [ ] **Step 6a: Replace `buildPawnDirs` implementation**

Replace the current method body:
```typescript
private buildPawnDirs() {
  const order = [...this.room.state.chess.playerOrder] as string[]
  // 2P: white (idx 0) bottom = -1, black (idx 1) top = +1
  // 3-4P: corner order bottom-left, bottom-right, top-right, top-left
  const dirs = order.length <= 2 ? [-1, 1] : [-1, -1, 1, 1]
  order.forEach((id, idx) => {
    this.pawnDirs[id] = dirs[idx % dirs.length] ?? -1
  })
}
```

private getPieceSymbol(piece: ChessPiece): string {
  if (piece.pieceType === "king") {
    const player = this.room.state.players.get(piece.ownerId)
    if (player?.characterId) {
      const ch = CHARACTERS.find(c => c.id === player.characterId)
